0. はじめに

この文書は、OpenPNE バージョン2.6.x から バージョン2.8.x へのアップグレード手順を書いたものです。

アップグレード作業を始める前に、データベースのバックアップを保存しておくことを強くおすすめします。

1. 手順

アップグレードは以下のような手順で進めます。

  1. アップグレードSQLの実行
  2. ソースコードの設置
  3. 設定ファイルの変更

2. 作業方法

2-1. アップグレードSQLの実行

使用している MySQL のバージョンに合わせて、アップグレードSQLを実行します。

2-1-1. MySQL 4.1 以降 (5.x 含む) の場合

  • OPENPNE_DIR/setup/sql/mysql41/upgrade/upgrade-2.6to2.8.sql
[実行例] 
$ cd setup/sql/mysql41/upgrade/
$ mysql -u username -p --default-character-set=utf8 DBNAME < upgrade-2.6to2.8.sql

2-1-2. MySQL 4.0 の場合

  • OPENPNE_DIR/setup/sql/mysql40/upgrade/upgrade-2.6to2.8.sql
[実行例] 
$ cd setup/sql/mysql40/upgrade/
$ mysql -u username -p DBNAME < upgrade-2.6to2.8.sql

2-2. ソースコードの設置

Webサーバ上に OpenPNE 2.8 のソースファイルを設置します。

新旧ソースファイル混在によるトラブルを防ぐため、上書きではなくソースコード全てを交換してください。 交換後は、OPENPNE_DIR/var ディレクトリの書き込み権限を適切なもの(例: 777)に変更してください。

『OpenPNE セットアップガイド』も参考にしてください。

2-3. 設定ファイルの変更

OpenPNE 2.6 の設定ファイルを参考にして 2.8 用の設定ファイルを作成します。

以上でアップグレード作業は完了です。
